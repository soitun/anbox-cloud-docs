# 1.30.1

These release notes cover new features and changes in Anbox Cloud 1.30.1.

Anbox Cloud 1.30.1 is a patch release. To understand minor and patch releases, see {ref}`Release notes <ref-release-notes>`.

Please see {ref}`Component versions <ref-component-versions>` for a list of updated components.

## Requirements

See the {ref}`Requirements <ref-requirements>` for details on general and deployment specific requirements to run Anbox Cloud.

## New features & improvements

- The Android WebView has been updated to 150.0.7871.64.

## Deprecations and removed functionality

There are no deprecations or removal of support for features in this release.

## Bug fixes

- Virtualized Android 16 images (`resolute:*-cf:*`) [failed to start](https://discourse.ubuntu.com/t/virtualized-android-16-images-fail-to-start-in-1-30-0-due-to-a-mesa-2404-snap-update/84624) due to a `mesa-2404` snap update that changed its internal library layout. The `anbox-runtime` snap now correctly handles the updated layout.
- Auto compaction of etcd from AMS has been removed, as this can be configured directly on the etcd server. To replicate the compaction triggered by AMS internally, the recommended configuration is to set the compaction period to 5 minutes.
- [LP 2141742](https://bugs.launchpad.net/anbox-cloud/+bug/2141742) TLS identities of internal services don't have a proper name.
- [LP 2157591](https://bugs.launchpad.net/anbox-cloud/+bug/2157591) The VHAL capability is not reported within the default timeout in new AAOS images.
- [LP 2158859](https://bugs.launchpad.net/anbox-cloud/+bug/2158859) Instance deletion operations time out under certain conditions.
- [LP 2160081](https://bugs.launchpad.net/anbox-cloud/+bug/2160081) Downloading an artifact from an instance fails because the fingerprint of the downloaded artifact doesn't match the expected one.
- [LP 2141916](https://bugs.launchpad.net/anbox-cloud/+bug/2141916) `anbox-cloud-appliance init` does not clean up in case of an error.
- [LP 2158053](https://bugs.launchpad.net/anbox-cloud/+bug/2158053) The Terraform scripts in 1.30.0 referenced `etcd` and `grafana-agent` instead of `charmed-etcd` and the OpenTelemetry Collector.
- [LP 2150702](https://bugs.launchpad.net/anbox-cloud/+bug/2150702) Private bug
